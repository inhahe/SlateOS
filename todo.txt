## Completed
- Per-CPU scheduler lock refactor: DONE (commit 449e218).  Split global
  SCHED lock into per-CPU spinlocks inside PerCpuScheduler + separate
  global task table lock.  timer_tick() no longer needs the global lock
  on the fast path.  Boot-tested SMP=1 and SMP=2.


## Known Bugs / Limitations

- Init task null-pointer crash on SMP: Intermittent page fault at
  RIP=0xffffffff80136b00, address=0x8 (null pointer + field offset 8),
  in task "init" on CPU 1.  Race condition — occurs ~50% of 2-CPU boot
  tests.  Likely a process/thread lifecycle race where shared state is
  accessed after another CPU deallocates it.  Not triggered on 1-CPU.
  Affects: init process system (kernel-ipc/kernel-process zone).

- Debug kernel binary too large for Limine: The debug build (~9MB) causes
  Limine to panic "High memory allocator: Out of memory" with 128M QEMU.
  Workaround: use release builds for boot testing, or increase QEMU memory
  to 256M+.  Long-term: strip debug sections or use split-debuginfo.

- PI futex: transitive priority inheritance — scheduler-side infrastructure
  is DONE (blocked_on_pi_addr field, pi_chain_boost(), cycle detection,
  depth limit, tested).  Waiting on kernel-ipc zone to wire it up in
  futex_lock_pi/futex_unlock_pi (see cross-zone request below).

## Cross-Zone Requests

- drivers: virtio-blk driver only discovers the FIRST PCI device
  (pci::find_device returns first match).  QEMU can present multiple
  virtio-blk devices (disk.img=vda, ext4_test.img=vdb, swap.img=vdc)
  but only vda is registered.  Need pci::find_all_devices() or a
  scan loop so all virtio-blk devices become available.  Blocking
  ext4 boot-time self-tests (ext4_test.img at build/ext4_test.img).

- (FIXED) kernel-core: CRC32C self-test was failing because the
  expected value for 32 zero bytes was wrong (0xAA36918A is a
  byte-swapped version of the correct value 0x8A9136AA).  Verified
  against Python reference implementation.  Our CRC32C algorithm was
  correct all along — ext4 metadata checksums ARE Linux-compatible.
  Combined with the journal byte order fix, ext4 compatibility is
  now fully resolved.

- kernel-ipc: Wire SYS_CLOCK_MONOTONIC (syscall/handlers.rs) to use
  hpet::elapsed_ns() when HPET is available, falling back to the APIC tick
  counter if not.  The HPET gives 10 ns resolution vs 10 ms from the APIC
  timer.  hpet::is_available() and hpet::elapsed_ns() are the public API.

- kernel-ipc: Wire SYS_SLEEP (syscall/handlers.rs) to use HPET-based
  timing for sub-10ms sleep accuracy.  Currently rounds up to 10 ms ticks.

- kernel-ipc: Store exit code from SYS_EXIT (handlers.rs:131) in PCB
  state so SYS_PROCESS_WAIT can retrieve it.  Currently the exit code
  argument is accepted but discarded.

- kernel-ipc: Fix mmap partial failure recovery (handlers.rs:500, 548).
  If mmap region partially maps then alloc_frame fails, already-mapped
  frames leak.  FIX AVAILABLE: replace the manual alloc-map loop with
  `mm::page_table::map_committed_range(pml4_phys, base_virt, num_frames, flags)`
  which does alloc+zero+map atomically with full rollback on partial failure.
  For MMIO path, use a similar pattern with `unmap_committed_range()` for cleanup.

- kernel-ipc: Verify SYS_PROCESS_ID (handlers.rs:1549) returns the
  owning process PID, not just the task ID.  Has a TODO comment.

- kernel-ipc: Wire transitive priority inheritance into futex_lock_pi /
  futex_unlock_pi (ipc/futex.rs).  The scheduler-side infrastructure is
  ready in sched/mod.rs and sched/task.rs:
    - `Task.blocked_on_pi_addr: Option<u64>`    set before blocking, clear on acquire
    - `sched::set_blocked_on_pi_addr(task_id, addr)` — setter
    - `sched::get_blocked_on_pi_addr(task_id)` — getter
    - `sched::pi_chain_boost(start_owner, donor_prio, find_owner)` — walk chain
    - `task::PI_CHAIN_DEPTH_LIMIT` (10) — max chain depth
  In futex_lock_pi: after `sched::boost_priority(owner, caller_prio)`,
  call `sched::pi_chain_boost(owner, caller_prio, |addr| pi_table.find_owner(addr))`.
  Before blocking: `sched::set_blocked_on_pi_addr(caller_task, Some(futex_addr))`.
  After acquiring: `sched::set_blocked_on_pi_addr(caller_task, None)`.
  Self-test already passes for the sched-side; needs futex-side integration test.

- (DONE) kernel-core: Hooked fs::cache::try_flush_expired() into the
  timer softirq (softirq.rs handle_timer).  Runs every 500 ticks (~5s)
  on CPU 0 only.  Uses try_lock to avoid deadlock in softirq context —
  if the cache lock is held, the flush is skipped and retried next cycle.
  Added try_flush_expired() non-blocking variant to fs/cache.rs.

- fs: (RESOLVED) Root mount now uses FAT on virtio-blk device.
  The FAT driver auto-mounts at "/" on boot, init deploys binaries fine.

## Known Limitations (fs zone)

- ext4 parse_dir_entries and volume name parsing use from_utf8_lossy(),
  which violates CLAUDE.md ("silent data corruption").  Paths should be
  treated as opaque bytes, not forced to UTF-8.  Fixing requires changing
  the VFS DirEntry name type from String to Vec<u8> or OsStr, then
  propagating through all VFS consumers.  Deferred until the VFS can be
  refactored — affects driver.rs:3049 and superblock.rs:286.

- (FIXED) ext4 journal byte order: converted all jbd2 field reads/writes
  from little-endian to big-endian (network byte order) per jbd2 spec.
  Added read_be32/read_be64/write_be32 helpers, replaced read_struct_pub
  with manual BE parsing in open(), scan_revokes(), replay_with_revokes().
  Write paths (write_block_header, commit descriptor tags,
  write_journal_superblock) now use to_be_bytes.  Journal is now
  compatible with Linux-formatted ext4 images.
