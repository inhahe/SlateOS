## Known Bugs / Limitations

- (FIXED) Workqueue worker stack overflow (#DF): Task 61 ("kworker") double-
  faulted during workqueue::self_test() with RSP 0x180 bytes below stack
  bottom (33152/32768 bytes used). Debug builds have unoptimized frames.
  Fix: increased TASK_STACK_SIZE from 2 frames (32 KiB) to 4 frames (64 KiB)
  and updated mm::kstack::STACK_FRAMES to match.

- (FIXED) Init process null-deref after BOOT_OK: Was caused by the same
  BTreeMap corruption as below (use-after-free via raw pointers during
  context switch).  Fixed by Box<Task> stable-address approach.  Init
  process now runs successfully on SMP, verified in boot-test serial output.

- (FIXED) Init task BTreeMap corruption on SMP: Was a use-after-free via
  raw pointers extracted from the task BTreeMap during context switch.
  After dropping the SCHED lock, concurrent BTreeMap operations on other
  CPUs could rebalance the tree, invalidating the pointers.  Fix: changed
  `BTreeMap<TaskId, Task>` to `BTreeMap<TaskId, Box<Task>>` so tasks have
  stable heap addresses independent of BTree node layout.

- Debug kernel binary too large for Limine: The debug build (~9MB) causes
  Limine to panic "High memory allocator: Out of memory" with 128M QEMU.
  Workaround: use release builds for boot testing, or increase QEMU memory
  to 256M+.  Long-term: strip debug sections or use split-debuginfo.

- (DONE) PI futex: transitive priority inheritance fully wired.
  futex_lock_pi now calls pi_chain_boost() after initial boost,
  sets blocked_on_pi_addr before blocking, clears after acquiring.
  find_pi_owner() callback looks up owners from PI_FUTEX_TABLE.

## Known Limitations (fs zone)

- ext4 parse_dir_entries and volume name parsing use from_utf8_lossy(),
  which violates CLAUDE.md ("silent data corruption").  Paths should be
  treated as opaque bytes, not forced to UTF-8.  Fixing requires changing
  the VFS DirEntry name type from String to Vec<u8> or OsStr, then
  propagating through all VFS consumers.  Deferred until the VFS can be
  refactored — affects driver.rs:3049 and superblock.rs:286.

- (FIXED) ext4 journal byte order: converted all jbd2 field reads/writes
  from little-endian to big-endian (network byte order) per jbd2 spec.
  Added read_be32/read_be64/write_be32 helpers, replaced read_struct_pub
  with manual BE parsing in open(), scan_revokes(), replay_with_revokes().
  Write paths (write_block_header, commit descriptor tags,
  write_journal_superblock) now use to_be_bytes.  Journal is now
  compatible with Linux-formatted ext4 images.

- (DONE) kernel-process: Wire rmap::add() into proc/pcb.rs try_resolve_fault().
  Added crate::mm::rmap::add(phys_frame.addr(), pml4_phys, frame_base) after
  successful map_frame.  Compaction can now find and migrate demand-paged
  user frames.

- (DONE) kernel-ipc: Wire scfilter::check() into syscall/dispatch.rs.
  Added scfilter check between bounds check and handler lookup.  Denied
  syscalls return PermissionDenied without invoking the handler.  Container
  syscall sandboxing is now fully enforced.

- (DONE) kernel-process: Wire cgroup memory controller into demand paging.
  Added try_charge_current_mem(1) before frame allocation in try_resolve_fault().
  On alloc failure, uncharges the pre-charged frame.  Cgroup memory limits
  are now enforced on user-space demand-paged allocations.  (Note: uncharge
  on free is handled by the per-frame cgroup tracking in frame.rs — the
  frame allocator calls uncharge_cgroup_free() on every free_frame() call.)

- (DONE) net zone: Integrate netns module with net/ globals.
  Added namespace-aware accessors to interface.rs (ns_ip, ns_info, ns_is_up)
  that query netns::interface_config() for non-root namespaces and fall back
  to the global IFACE for root NS (0).  configure() now syncs to netns root NS
  when netns is initialized.  Added send_ns() to ipv4.rs which uses the
  namespace's IP as source and netns::route_lookup() for next-hop routing.
  send() delegates to send_ns(ROOT_NS, ...).  ARP cache and firewall remain
  global (documented as shared / future per-namespace).  Added is_initialized()
  to netns.rs for boot-order safety (net::init runs before netns::init).
  Per-namespace firewall rules: DONE (commit 8f1ec5a).  16 rules + 32
  conntrack entries per namespace, ns_init/ns_enable/ns_add_rule API,
  ipv4::send_ns() uses check_outbound_ns().  3 self-tests.
  Remaining work: per-namespace ARP cache (blocked on veth pairs),
  and wiring process→container→net_ns lookup into socket-layer callers
  (requires changes in kernel-process zone for PCB net_ns field and
  kernel-ipc zone for syscall handler changes — cross-zone work).

## POSIX Layer Notes

- (DONE) CWD tracking: Per-process userspace CWD buffer (4096 bytes) in
  posix/src/unistd.rs.  chdir() validates via SYS_FS_STAT + is_dir(),
  stores normalized absolute path.  getcwd() copies from buffer.
  resolve_path() public API resolves relative paths against CWD; wired
  into all path-taking file operations (open, stat, lstat, unlink, rename,
  link, symlink linkpath, readlink, mkdir, rmdir, truncate, access,
  opendir, posix_spawn, execve).  Path normalization handles ".", "..",
  redundant "/", and trailing slashes.  20 unit tests for normalize_path.

- (FIXED) Pipe/socket dup: dup(pipe_fd) and dup(socket_fd) now work.
  Implemented fd-level handle refcounting via is_handle_referenced() scan
  in fdtable.rs.  close() skips kernel handle close when other fds still
  reference it.  Socket metadata (SocketMeta) is copied on dup for
  getpeername/getsockname support.  Shell pipelines (cmd1 | cmd2) can
  now properly redirect pipe endpoints.

- (FIXED) posix_spawn/execve now use mmap for dynamic ELF buffer allocation.
  stat() gets file size, mmap allocates exactly that, munmap frees after
  spawn/exec.  No longer limited to 512 KiB.

- (FIXED) Socket dup now works.  Same refcounting fix as pipe dup
  (is_handle_referenced scan + copy_meta for socket metadata).

- poll/select report all valid fds as ready (optimistic readiness).  This
  works correctly for files (POSIX says always ready) and is conservative
  for pipes/sockets (programs handle blocking on I/O).  A proper kernel
  event notification mechanism (eventfd/completion ports) would allow
  accurate readiness reporting.

- getaddrinfo() uses static storage (no heap) and returns at most one
  result.  Not re-entrant — same as POSIX gethostbyname().

- ioctl TCSETS/TCGETS are accepted but don't actually change terminal
  behavior (no line discipline).  Programs that set raw mode work via
  VT100 escape sequences.  Will need real terminal mode support when
  porting interactive programs (vim, less, etc.).

- Unit tests exist in #[cfg(test)] modules but can't run on the host
  because the crate targets x86_64-unknown-none.  Need a QEMU-based
  test runner or a separate host-testable crate that extracts pure
  logic functions.
