you: how would i use two different accounts in two different sessions? it seems that claude code and my own vibecoded
     alternative use some kind of global claude auth
claude: The easiest approach is the `CLAUDE_CONFIG_DIR` environment variable. Each value points to a separate config
        directory with its own auth:

        ```bash
        # Terminal 1 — account A
        set CLAUDE_CONFIG_DIR=%USERPROFILE%\.claude-account-a
        claude

        # Terminal 2 — account B
        set CLAUDE_CONFIG_DIR=%USERPROFILE%\.claude-account-b
        claude
        ```

        First time you run each, it'll ask you to log in. After that, each terminal uses its own saved credentials. Both
        can run simultaneously in separate terminals against the same codebase.

        You could make it even simpler with batch files or shell aliases so you just type `claude-a` or `claude-b`.

        There are also third-party tools like [claude-swap](https://github.com/realiti4/claude-swap) and [CCS (Claude
        Code Switch)](https://ccs.kaitran.ca/) that wrap this into a nicer workflow, but the environment variable
        approach is all you really need.
